Bashrc Startup Speed

Implementation plan: 3 strategies, 8 files, ~0.5s → ~0.2s

Three Optimization Strategies

Strategy 1
Replace $(uname) with $OSTYPE
Eliminate 10 subshell forks across 6 files. $OSTYPE is a bash built-in variable — zero cost, no fork needed.
-0.07s
Strategy 2
Eliminate fork+eval overhead
Replace mise activate with inline shims PATH. Inline direnv hook with cached binary path. No subprocess spawning at startup.
-0.24s
Strategy 3
Defer aliases to first prompt
Move alias_available PATH lookups from startup to PROMPT_COMMAND. Aliases are ready before user types anything.
-0.17s

Key Before/After: mise.bash

Before (0.201s)
#!/usr/bin/env bash

_setup_mise() {
  if exists mise; then
    if [ "$(uname)" = "Darwin" ]; then
      export MISE_ASDF_COMPAT=1
    fi
    eval "$(mise activate bash)"
  fi
}
_setup_mise
unset -f _setup_mise
After (0.000s)
#!/usr/bin/env bash

if exists mise; then
  if [[ $OSTYPE == darwin* ]]; then
    export MISE_ASDF_COMPAT=1
  fi
  export PATH="$HOME/.local/share/mise/shims:$PATH"
fi

Key Before/After: aliases (deferred)

Before (0.170s at startup)
_setup_aliases() {
  alias_available cat bat
  alias_available curl curlie http
  ...
}

_setup_aliases
unset -f _setup_aliases
After (0.000s at startup, 0.170s at first prompt)
_setup_aliases() { ... }

_deferred_aliases() {
  _setup_aliases
  unset -f _setup_aliases \
    _deferred_aliases alias_available
  # Remove ourselves from PROMPT_COMMAND
  PROMPT_COMMAND="${PROMPT_COMMAND//_deferred_aliases;/}"
}
PROMPT_COMMAND="_deferred_aliases;${PROMPT_COMMAND}"

Edge Cases and Considerations

mise env vars not auto-loaded on cd
Shims mode doesn't run mise hook-env. Per-directory env switching is handled by direnv instead. Tool versions still resolve correctly via shims.
direnv changes hook format
The inlined hook matches direnv hook bash output which has been stable for years. Binary path is cached in _direnv_bin and resolves via command -v.
PROMPT_COMMAND is an array (Bash 5.1+)
The direnv hook checks declare -p and handles both string and array forms. The deferred aliases use string form which matches the existing history.bash pattern.
$OSTYPE value varies across OS versions
Using glob matching: darwin* matches darwin23, darwin24, etc. linux-gnu* matches all Linux variants.

Trade-offs

Decision Benefit Cost
mise: shims vs activate
Eliminates 0.2s startup + 20ms per-prompt hook-env
~5ms overhead per tool invocation via shim resolution
direnv: inline vs eval
Eliminates 0.045s fork at startup
Must update manually if direnv changes hook format
aliases: defer vs immediate
Moves 0.17s of PATH lookups off critical startup path
Aliases resolve at first prompt instead of at source time
$OSTYPE vs $(uname)
Zero-cost built-in variable, no fork
Pattern matching syntax differs from string equality
